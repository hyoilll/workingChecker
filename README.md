# workingChecker

退勤前の1日の業務振り返りを、AIエージェントとの対話を通じて行うための個人ワークスペースです。

## 特徴

- **コード0行**。AIエージェント（Copilot CLI など）のルール定義だけで成立
- ルールはすべて [`AGENTS.md`](./AGENTS.md) に記述
- 振り返りは `logs/YYYY-MM-DD.md` に Markdown で保存される
- AIが動的に深掘り質問をするため、毎日違う切り口で内省できる

## 使い方

1. Copilot CLI をこのディレクトリで起動
   ```sh
   cd workingChecker
   copilot
   ```

2. 振り返りを始めたいとき、AIに次のように話しかける
   - 「振り返り始めて」
   - 「今日の振り返り」
   - 「retro」

3. AIが業務・苦労・工夫について深掘り質問を投げてくるので、対話形式で回答する

4. 最後に要約が提示されるので、内容を確認して保存する
   - 保存先: `logs/YYYY-MM-DD.md`
   - 既存ファイルがある場合は上書き / 追記 / キャンセルを選べる

## ディレクトリ構成

```
workingChecker/
├── AGENTS.md              # AIへの振り返り進行ルール（核心）
├── README.md              # このファイル
├── templates/
│   └── reflection.md      # 保存Markdownのテンプレート
└── logs/                  # 日々の振り返りが保存される
    └── YYYY-MM-DD.md
```

## 必ずカバーされる3項目（各業務につき）

AIは毎日、**今日行った業務の数だけミニループ**を回し、各業務について以下を必ず聞いてきます：

1. **業務の概要**
2. **その業務で苦労したこと・大変だったこと**
3. **その業務で工夫したこと・学んだこと**

業務が1つだけの日は1ループ、3つあれば3ループ。各ループは 3〜6 ターン程度の深掘り対話になります。

加えて、文脈に応じて以下の観点から動的に深掘りします：

- 具体化 / 理由・背景 / 感情 / 代替案 / 学びの一般化 / 再発防止

## カスタマイズ

- **質問の観点を追加したい** → `AGENTS.md` の「5. 深掘りの観点」を編集
- **Markdown の見た目を変えたい** → `templates/reflection.md` を編集
- **トリガー語を増やしたい** → `AGENTS.md` の「1. トリガー」に追記

## 注意

- ログは Git 管理していません（プライベートな内省のため）
- Markdown ファイルは通常のエディタで後から編集できます
