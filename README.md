# workingChecker

1日の出来事（業務・日常問わず）を、AIエージェントとの対話を通じて振り返るための個人ワークスペースです。

## 特徴

- **コード0行**。AIエージェント（Claude Code）のルール定義だけで成立
- ルールはすべて [`CLAUDE.md`](./CLAUDE.md) に記述（Claude Code が自動読み込み）
- 振り返りは `logs/YYYY-MM-DD.md` に Markdown で保存される
- AIが動的に深掘り質問をするため、毎日違う切り口で内省できる

## 使い方

1. Claude Code をこのディレクトリで起動
   ```sh
   cd workingChecker
   claude
   ```

2. 振り返りを始めたいとき、AIに次のように話しかける
   - 「振り返り始めて」
   - 「今日の振り返り」
   - 「retro」
   - 「日記」
   - 「今日の記録」

3. AIが出来事・苦労・工夫について深掘り質問を投げてくるので、対話形式で回答する

4. 最後に要約が提示されるので、内容を確認して保存する
   - 保存先: `logs/YYYY-MM-DD.md`
   - 既存ファイルがある場合は上書き / 追記 / キャンセルを選べる

## ディレクトリ構成

```
workingChecker/
├── CLAUDE.md              # AIへの振り返り進行ルール（核心）
├── AGENTS.md              # 他AIツール向けの参照案内
├── README.md              # このファイル
├── templates/
│   └── reflection.md      # 保存Markdownのテンプレート
└── logs/                  # 日々の振り返りが保存される
    └── YYYY-MM-DD.md
```

## 必ずカバーされる3項目（各出来事につき）

AIは毎日、**今日あった出来事の数だけミニループ**を回し、各出来事について以下を必ず聞いてきます：

1. **出来事の概要**
2. **その出来事で苦労したこと・大変だったこと**
3. **その出来事で工夫したこと・学んだこと**

出来事が1つだけの日は1ループ、3つあれば3ループ。各ループは 3〜6 ターン程度の深掘り対話になります。

加えて、文脈に応じて以下の観点から動的に深掘りします：

- 具体化 / 理由・背景 / 感情 / 代替案 / 学びの一般化 / 再発防止

## カスタマイズ

- **質問の観点を追加したい** → `CLAUDE.md` の「5. 深掘りの観点」を編集
- **Markdown の見た目を変えたい** → `templates/reflection.md` を編集
- **トリガー語を増やしたい** → `CLAUDE.md` の「1. トリガー」に追記

## 注意

- リポジトリは GitHub でリモート管理
- 振り返り保存時にAIが自動で `git add logs/YYYY-MM-DD.md` → `git commit -m "reflect: YYYY-MM-DD"` を実行
- Markdown ファイルは通常のエディタで後から編集できます（手動で再コミットを）
